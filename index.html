Settings

TURN Server (optional)

In strict NAT or corporate network environments where the default STUN-based connection fails, a TURN relay server lets the app connect via a relay. Leave blank to use the default configuration (STUN only).

Custom PeerJS Server (optional)

By default, Quick Pair uses the public PeerJS cloud signalling server. Configure a self-hosted PeerJS server here if you prefer not to rely on the public server. Changes take effect on the next connection attempt.
